# Brand System

## Essencia da marca

Flow Merge nao deve soar como SaaS generico de marketing.

Ele deve parecer:

- tecnico
- controlado
- operacional
- inteligente
- legivel sob pressao

Palavras-chave:

- command center
- operator surface
- local-first
- graph-native
- runtime-aware
- explanatory workflows

## Personalidade

- direto, sem hype vazio
- denso, mas legivel
- confiante sem exagero
- mais proximo de ferramenta de operacao do que de site corporativo

## Tom de voz

### Fazer

- falar de operacao, contexto, decisao, impacto, fluxo
- usar verbos concretos: executar, observar, comparar, reagir, ajustar
- descrever sistemas em vez de promessas abstratas

### Evitar

- jargoes vagos de SaaS
- claims sem mecanismo
- frases que poderiam estar em qualquer startup de IA
- excesso de superlativos

## Regras de copy

- headline curta e forte
- subtitulo explicando o mecanismo
- cada secao com uma funcao so
- CTA ligado a acao real

## Linguagem visual

O visual deve nascer do canvas do produto.

### Elementos-base

- fundo escuro tecnico
- grid visivel
- paineis com borda fina
- nodes como unidade de narrativa
- conectores e trilhas indicando causalidade
- tipografia compacta, sem parecer apertada

### Semantica de cor

- azul: core e manipulacao do fluxo
- azul claro: analytics e leitura
- verde: visualizacao, saude, resultado
- amarelo: trigger, monitoramento, atencao
- vermelho: falha, risco, erro
- cinza tecnico: estrutura, chrome e areas neutras

## UI Principles

### 1. O marketing deve parecer produto

A landing nao deve se comportar como site tradicional com hero isolado, cards genericos e secoes de marketing padrao.

Ela deve parecer uma extensao do Flow Merge.

### 2. A narrativa deve acontecer em nodes

O produto se explica melhor quando:

- eventos entram
- nodes processam
- um insight aparece
- um operador entende o que fazer

Essa estrutura precisa aparecer na landing.

### 3. Menos paginas, mais superficie

Quando possivel, a UX deve parecer uma superficie unica.

### 4. O login e parte da experiencia

O acesso local nao e um modal perdido. Ele faz parte da narrativa do produto porque reforca:

- desktop-first
- local-first
- espaco privado de operacao

### 5. Densidade com hierarquia

Flow Merge pode ser denso, mas nunca confuso.

Regras:

- um plano dominante por secao
- uma leitura primaria clara
- detalhes secundarios so quando agregam

## Movimento

Motion deve sugerir sistema vivo, nao decoracao gratuita.

Prioridades:

- entrada suave da topbar
- revelacao progressiva dos nodes
- conexoes com pulso sutil
- hover que indique relacao e foco

Evitar:

- microanimacoes vazias
- bouncing
- gradientes decorativos sem funcao

## Componentes narrativos

Na comunicacao visual, os principais blocos sao:

- topbar estilo app
- canvas board
- info node
- flow rail
- metric tile
- compare strip
- login dock

## Regras para futuras telas

Uma tela nova de marketing, onboarding ou docs visuais deve responder:

1. Isso parece uma extensao do canvas?
2. O usuario entende o tipo de produto nos primeiros segundos?
3. O layout privilegia leitura operacional e nao ornamentacao?
4. Os nodes contam uma historia real de uso?
5. O design reforca a tese "workflow que se explica sozinho"?
